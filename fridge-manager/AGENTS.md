# AGENTS.md

## 목적
이 저장소의 목표는 냉장고 음식 관리 Web App MVP를 구현하는 것이다.

## 먼저 읽을 문서
작업 시작 전 반드시 아래 문서를 읽는다.
1. `docs/project-overview.md`
2. `docs/requirements.md`
3. `docs/tech-stack.md`

## 작업 원칙
* 모든 구현은 `docs/requirements.md`를 기준으로 한다.
* MVP 범위를 넘는 기능은 추가하지 않는다.
* 기존 코드 스타일과 구조를 우선 따른다.
* 변경은 작고 명확하게 유지한다.
* 불필요한 리팩토링과 라이브러리 추가를 피한다.

## 구현 우선순위
1. 프로젝트 기본 구조
2. Tailwind 설정
3. 타입 정의
4. localStorage 유틸
5. 목록 UI
6. 추가/수정/삭제
7. 정렬/검색/유통기한 상태 표시

## 완료 기준
* 요구사항에 있는 기능만 구현되어야 한다.
* TypeScript 오류가 없어야 한다.
* 주요 UI가 PC/모바일에서 자연스럽게 동작해야 한다.
* 변경 사항이 작동하고 기존 동작을 깨지 않아야 한다.

## 문서 갱신 규칙
- 작업 전 `AGENTS.md`, `ARCHITECTURE.md`, `docs/`의 관련 문서를 확인한다.
- 코드 변경이 설계, 구조, 요구사항, 사용 방법에 영향을 주면 관련 문서를 함께 수정한다.
- 문서 변경이 필요 없으면 문서를 억지로 수정하지 않는다.
- `AGENTS.md`는 자주 바꾸지 말고, 저장소 전반 규칙이 바뀔 때만 수정한다.
- `ARCHITECTURE.md`는 실제 구조 변경이 있을 때만 수정한다.
- `docs/`는 기능/요구사항/구현 방식이 바뀐 문서만 갱신한다.
- 작업 완료 전, 문서 업데이트 필요 여부를 반드시 점검한다.
