# 프로젝트 요구사양
## 1. 음식물 추가 기능
- 필수로 입력되어야 하는 항목
    - 음식물의 이름
    - 보관 위치
        - 냉장실
        - 냉동실
    - 등록일
        - 기본값은 현재 날짜로 설정되며 사용자가 변경할 수 있다
        - 날짜 포맷은 yyyy-mm-dd
- 선택적으로 입력 가능한 항목
    - 유통기한
    - 수량
    - 메모

## 2. 음식물 조회 기능
- 냉장실과 냉동실로 나누어 음식물 목록 표시
- 정렬 기능
    - 유통기한 빠른 순
    - 최근 등록 순
- 유통기한 지난 음식 강조 표시
- 유통기한 임박 음식 강조 표시
    - 3일 이하 남으면 경고
- 음식물 검색 기능
    - 음식물의 이름으로 검색
- 남은 일수 계산

## 3. 음식물 관리 기능
- 음식물을 선택하면 아래 정보를 수정할 수 있다.
    - 이름
    - 보관 위치
    - 등록일
    - 유통기한
    - 수량
    - 메모
- 음식물 삭제

# 4. UI 요구사양
- 반응형 웹으로 만든다. PC와 모바일에서 모두 자연스럽게 표시되어야 한다.
- 메인 페이지에서 음식 목록을 한눈에 볼 수 있어야 한다. 단, 냉장실과 냉동실을 구분해서 표시해야 한다.
- 음식 추가/수정은 모달 또는 별도 폼 페이지로 구현한다.
- Tailwind CSS로 깔끔하고 직관적인 UI를 만든다.
- 상태 배지와 유통기한 경고 색상을 적용한다.

# 5. 품질 요구사항
- TypeScript 타입을 명확히 작성한다.
- 컴포넌트를 너무 크게 만들지 말고 적절히 분리한다.
- 폼 유효성 검사를 포함한다.
- 에러 처리와 로딩 상태를 포함한다.
- 유지보수하기 쉬운 폴더 구조를 사용한다.
